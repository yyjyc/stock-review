## 复盘引导功能设计方案

### 1. 新增页面和路由

* 创建 `/review-guide` 路由，作为复盘引导入口页面

### 2. 页面布局设计

**步骤引导区域**：

* 使用 Element Plus 的 Steps 组件展示4个步骤

* 每个步骤显示完成状态（待完成/进行中/已完成）

**步骤内容区域**：

* 根据当前步骤嵌入对应的模块内容

* 每个步骤完成后点击"下一步"进入下一环节

**4个步骤**：

1. 活跃市值复盘 - 录入当日活跃市值数据
2. 今日操作复盘 - 处理待实施的交易计划
3. 持仓情况复盘 - 检查持仓状态和预警
4. 选股模块 - 添加新的选股计划

### 3. 汇总页面设计

完成所有步骤后显示汇总页面，包含：

* 今日活跃市值状态

* 今日已执行的操作列表

* 当前持仓概览（总金额、总盈亏）

* 新增的选股计划

* 日期时间戳

### 4. 图片导出功能

* 使用 `html2canvas` 库将汇总页面转换为图片

* 提供下载按钮导出 PNG 图片

### 5. 技术实现

* 使用 localStorage 存储当日复盘进度

* 使用 provide/inject 在步骤间共享数据

* 安装 html2canvas 依赖用于图片导出
